# Открытые вопросы

Список решений, которые нужно принять до старта или в процессе разработки. Каждый вопрос привязан к фазе.

---

## К Фазе 0 (сейчас, до старта) — ✅ решено

### Multi-machine синхронизация
**Решение:** на старте без синхронизации. Пользователь импортирует royalty-файл отдельно на каждой машине. Одинаковые KDP-отчёты дают одинаковые цифры — повторный импорт безопасен.

End-to-end зашифрованная синхронизация (royalty шифруется ключом пользователя, blob лежит на Railway, decrypt на машине) — премиум-фича на будущее, +1–2 недели работы.

### CRA → Vite миграция
**Решение:** откладываем. CRA в Electron работает; миграция на Vite — отдельной фазой, когда станет очевидно, что время сборки мешает.

### Telegram-бот
**Решение:** не трогаем. Остаётся серверным как сейчас.

## К Фазе 1 (Backend cleanup)

### API-ключи: где генерируются
**Принято:** в «зонтике» — Railway-портале с лендингом и личным кабинетом. Пользователь покупает подписку, получает ключ, скачивает .dmg / .exe. Ключ привязан к учётке.

**Открыто:**
- **Сколько ключей на одну подписку?** Один (привязан к одному устройству, переустановка через «отозвать ключ → перевыдать») или несколько (для multi-machine)?
- **Как ключ сбрасывается?** Кнопка в личном кабинете «отозвать и выдать новый»?
- **Триал:** есть ли период без ключа? Если да — сколько дней? 7 / 14 / 30?

### Платёжная система
Варианты:

- **Stripe.** Самая удобная инфраструктура, требует юрлица в поддерживаемой стране. Комиссия ~3%. Сам считаешь и платишь налоги.
- **Paddle.** Merchant of Record (MoR) — берёт на себя НДС/налоги во всех странах. Удобно для international. Комиссия ~5%.
- **LemonSqueezy.** Аналог Paddle, чуть дешевле, поддерживает crypto. Комиссия ~5%.

**Открыто:** что выбрать. Зависит от того, где у тебя будет юрлицо и нужна ли поддержка мировых налогов на старте.

### Тарифная сетка
Варианты, которые встречаются у похожих SaaS для KDP-авторов:

- Один тариф $20–40/мес — все фичи
- Tiered: Basic / Pro / Team — по числу KDP-аккаунтов или объёму PPC-spend
- Lifetime deal $200–400 — как промо для первых пользователей

Решение влияет на:
- Логику API-ключей (один ключ или несколько)
- Ограничения в backend (rate-limits на PPC-запросы)
- Структуру таблиц `accounts` и `subscriptions`

## К Фазе 5 (Packaging + signing)

### Юрлицо
Для покупки EV-сертификата под Windows и для Stripe нужна регистрация юрлица.

Варианты:
- ИП в России — ~5 000 ₽, 2–7 дней
- LLC в США через Stripe Atlas — ~$500
- ИП в Эстонии через e-Residency — ~€100

**Открыто:** какое подходит твоей юрисдикции и налоговому профилю?

### Sentry или альтернатива
Sentry — стандарт. ~$26/мес базовый план достаточно для до 50K событий в месяц. Альтернативы: Bugsnag, Rollbar, self-hosted GlitchTip (бесплатно, но возиться).

**Решение по умолчанию:** Sentry developer plan.

### GitHub Releases приватный или публичный
Auto-updater качает обновления из GitHub Releases. Если репо публичный — любой может скачать .dmg / .exe и пользоваться без оплаты.

**Решение по умолчанию:** репо приватный, доступ к Releases через token, который вшит в Electron-приложение и проверяет API-ключ перед загрузкой обновления.
