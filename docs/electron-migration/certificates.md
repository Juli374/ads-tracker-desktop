# Code Signing — для macOS и Windows

## Зачем

Когда пользователь скачивает .dmg или .exe из интернета и пытается открыть, ОС хочет понять: это реально от того разработчика, которого ты считаешь? Не подменили ли по дороге? Это и проверяет цифровая подпись.

Без подписи поведение пугающее:

- **macOS:** «Приложение нельзя открыть, Apple не может проверить, нет ли вредоносного ПО». Кнопки «всё равно открыть» нет — нужно идти в Системные настройки → Безопасность и явно разрешить. ~80% пользователей сдаются.
- **Windows:** SmartScreen с синим экраном «Защитник Windows предотвратил запуск нераспознанного приложения». Кнопка «всё равно запустить» спрятана за «Подробнее». ~50% сдаются.

Для платного продукта это смерть: клиент платит и сразу натыкается на «вирус, не открывать».

---

## macOS

Только один путь: **Apple Developer Program**.

- Подписка $99/год, оплата картой на developer.apple.com
- Через ~24 часа после оплаты в личном кабинете доступен **Developer ID Certificate**. Скачиваешь, устанавливаешь в Keychain — electron-builder автоматически использует его при сборке.
- Дополнительно нужна **notarization**: после подписи готовая сборка отправляется в Apple, их сканер проверяет на вирусы и возвращает «штамп». Делается утилитой `notarytool`, входит в тот же $99/год. Без notarization подпись не принимается на свежем macOS.

Альтернатив нет — это монополия Apple. Регистрироваться можно как:

- **Individual** — проще: достаточно карты и Apple ID. Подходит для старта.
- **Organization** — требует D-U-N-S Number и юрлица, но даёт работу в команде и название компании в подписи вместо твоего имени.

**Рекомендация для старта:** Individual, $99/год. Через несколько месяцев можно проапгрейдить до Organization.

## Windows

Тут три уровня, отличаются ценой и тем, как быстро SmartScreen перестаёт ругаться.

### Без подписи — $0
SmartScreen ругается всегда. Не вариант для платного продукта.

### Standard Code Signing — ~$80–250/год
Поставщики: DigiCert, Sectigo, SSL.com.

После того как 1–3 тысячи пользователей скачают и установят сборку без проблем, Microsoft постепенно «прокачает репутацию» сертификата, и SmartScreen замолчит. Срок — обычно 1–3 месяца.

Минус: первые сотни клиентов всё равно увидят пугалку. Для платного запуска — медленный старт.

Плюс: Standard сертификат можно купить **на физлицо**, без юрлица.

### EV Code Signing (Extended Validation) — ~$300–500/год
Те же поставщики.

Никаких предупреждений с первого инсталлятора. Microsoft доверяет EV-сертификату сразу, потому что для его выдачи проводится строгая проверка юрлица.

Требует:
- Юрлицо (ИП или ООО), зарегистрированное на любую территорию
- Подтверждение адреса и телефона компании
- Иногда D-U-N-S Number (бесплатно, выдаёт Dun & Bradstreet)
- **Физический USB-токен (HSM)**, который тебе пришлют почтой — без него подписать нельзя

На физлицо EV-сертификат не выдают.

### Azure Trusted Signing — новинка от Microsoft, ~$10/мес
Самый удобный вариант 2026 года.

Никакого USB-токена, ключи хранятся в Azure HSM, подпись — через API. SmartScreen доверяет сразу, как EV-сертификату.

Требует:
- Учётка Azure
- Подтверждение юрлица через Microsoft Partner Center (3–7 дней)

Ограничение: для индивидуальных разработчиков (без юрлица) пока недоступно.

### Что выбрать

- Если есть юрлицо или готов открыть — **Azure Trusted Signing** (~$120/год). Лучшая цена + сразу работает.
- Если хочешь физический сертификат для контроля — **EV Code Signing** (~$300–500/год).
- Если юрлица нет и не планируется — **Standard Code Signing на физлицо**, мириться с тем, что первые 1–3 месяца SmartScreen ругается.

## Бюрократия — про юрлицо

Для Windows-сертификата (EV или Azure) понадобится открытое юрлицо. Это:

- **ИП в России** — ~5 000 ₽ госпошлина, 2–7 дней
- **LLC в США** (Wyoming / Delaware) через сервисы вроде Stripe Atlas — ~$500
- **ИП в Эстонии** через e-Residency — ~€100, удобно для онлайн-бизнеса с европейскими клиентами

Юрлицо потребуется и для Stripe / Paddle, поэтому это разовая инвестиция, которая окупит себя сразу через возможность принимать платежи.

## Когда покупать

**Не сейчас.** Сертификаты нужны только в **Фазе 5** (упаковка). До этого можно разрабатывать и тестировать без подписи — на своей машине Mac можно открыть unsigned-сборку через правый клик → «Открыть», на Windows — кнопкой «Всё равно запустить».

Покупать сертификаты примерно за 2–3 недели до планируемого первого релиза, чтобы успеть пройти верификацию (особенно для EV — там физическая доставка USB-токена, неделя минимум).
